# Изменения в проекте T-Bank Mock (март 2026)

## Исправленные проблемы

### 1. Добавление операций (history.py)
**Проблема:** При добавлении ручных операций через панель не создавалась полная структура данных.

**Решение:**
- Добавлены поля: `operationTime`, `merchant`, `counterparty`, `amount_data`, `signedAmount`, `debitAmount`, `creditAmount`
- Теперь операции имеют полную структуру, совместимую с шаблонами банка
- При добавлении операции автоматически генерируется PDF-чек (для расходов)

### 2. Генерация PDF-чеков (func.py, transfer.py)
**Проблема:** Отсутствовала функция генерации чеков для ручных операций.

**Решение:**
- В `func.py` добавлена функция `generate_operation_receipt()` - генерация чека по данным операции
- В `transfer.py` добавлена функция `generate_receipt_for_manual_op()` - интеграция с системой переводов
- В `panel_bridge.py` добавлен API endpoint `/api/operations/generate_receipt`
- В интерфейсе панели добавлена кнопка 📄 для генерации чека

### 3. Расчёт баланса (balance.py, history.py)
**Проблема:** Баланс не учитывал ручные операции корректно.

**Решение:**
- Функция `_effective_balance()` в balance.py использует `compute_manual_balance_adjustment()` из history.py
- Баланс динамически корректируется на сумму ручных операций за текущий месяц
- Исправлено дублирование расходов в `calculate_stats()` (убрано добавление `total_out_rub`)

### 4. Статистика доходов/расходов (panel_bridge.py, history.py)
**Проблема:** Статистика могла показывать некорректные данные.

**Решение:**
- `calculate_stats()` теперь правильно считает только операции за текущий месяц
- Исключены скрытые операции из статистики
- Гистограммы подменяются с учётом ручных операций

## Новые возможности

### Генерация чеков
- При добавлении новой операции (расход) чек генерируется автоматически
- Для существующих операций - кнопка "Чек" в списке операций
- Чеки сохраняются в формате PDF с автоматическим сжатием

### Полная структура операций
Каждая ручная операция теперь содержит:
```json
{
  "id": "m_xxx",
  "date": "ДД.ММ.ГГГГ, ЧЧ:ММ:СС",
  "amount": 10000.0,
  "type": "Debit/Credit",
  "description": "...",
  "bank": "Название банка",
  "title": "Имя получателя",
  "subtitle": "Категория",
  "phone": "+7...",
  "bank_preset": "vtb/sber/...",
  "operationTime": {"milliseconds": ..., "seconds": ...},
  "merchant": {"name": "...", "logo": "..."},
  "counterparty": {"name": "..."},
  "amount_data": {"value": ..., "currency": "RUB"},
  "signedAmount": -10000.0,
  "debitAmount": 10000.0,
  "creditAmount": 0
}
```

## API endpoints

### POST /api/operations/add
Добавление новой операции.
```json
{
  "direction": "out",
  "amount": 1000,
  "bank": "ВТБ",
  "title": "Иван И.",
  "bank_preset": "vtb",
  "datetime": "2026-03-22T10:00:00"
}
```
Ответ: `{"status": "ok", "id": "m_xxx", "receipt_path": "receipt_..."}`

### POST /api/operations/generate_receipt
Генерация чека для существующей операции.
```json
{"id": "m_xxx"}
```
Ответ: `{"status": "ok", "receipt_path": "receipt_..."}`

## Тестирование

1. Запустить mitmproxy с проектом
2. Открыть панель: http://85.209.135.247:8082/admin
3. Добавить операцию:
   - Указать сумму, банк, направление
   - Выбрать пресет банка (логотип)
   - Указать дату и время
4. Проверить:
   - Операция появилась в списке
   - Статистика обновилась
   - Чек сгенерирован (для расхода)
5. Проверить баланс в приложении банка - должен учитывать операцию
